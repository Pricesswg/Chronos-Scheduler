# Contributing to Chronos Scheduler

Thank you for your interest in contributing to Chronos Scheduler.

Chronos Scheduler is currently in active development, and community feedback is very useful to improve stability, detect issues, and refine the user experience.

There are two main ways to contribute to the project:

## 1. Test the application and report issues

One of the most useful ways to help is by testing Chronos Scheduler in real use cases.

If you find a bug, an unexpected behavior, or something that does not work as intended, please open an issue on GitHub.

When reporting an issue, please include as much useful information as possible:

1. A clear description of the problem
2. The steps needed to reproduce the issue
3. What you expected to happen
4. What actually happened
5. Screenshots or logs, if available
6. Information about your environment, such as browser, operating system, Home Assistant version, or installation method if relevant

Before opening a new issue, please check if the same problem has already been reported.

## 2. Contribute with code

Code contributions are welcome.

If you want to improve Chronos Scheduler, fix a bug, or add a new feature, please follow this process:

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes
4. Test your changes before submitting them
5. Open a Pull Request with a clear description of what you changed and why

Please do not submit unrelated changes in the same Pull Request. Keeping changes focused makes the review process easier and faster.

All code contributions will be reviewed before being merged into the main project. A Pull Request may be accepted, modified, or rejected depending on code quality, project direction, and compatibility with the existing structure.

## Pull Request guidelines

When opening a Pull Request, please include:

1. A short summary of the change
2. The reason for the change
3. Any issue that the Pull Request fixes, if applicable
4. Notes about testing performed

This Pull Request fixes an issue where scheduled actions were not correctly saved after editing an existing schedule.

Tested by creating, editing, saving, and reloading multiple schedules.
